## 1. 目的（出発点）
- 生活用品や旅行費を賄うため、日本の懸賞へ効率よく応募し当選率を最大化するシステムを作る。
- 当初は「Xでbotを作り完全自動で応募」を希望 → 調査の結果、完全自動は規約/凍結/API有料化で四重に塞がれていると判明。

## 2. 3本柱の方針（不変の原則）
1. **当選率最大化 × 安全（規約・法令遵守・凍結回避）× 低コスト（ゼロ円開始）** を同時に満たす。
2. プラットフォーム規約・各懸賞規約・robots.txt を遵守。**検出回避技術（指紋偽装・CAPTCHA突破・複数アカウント量産）は採用しない**。
3. 自動化できる工程（発見・収集・管理・通知・下書き・入力補助・許可された応募）と、人間が担う工程（規約上自動化不可な最終応募・購入・投函）を明確に分離。
4. 偽陽性より偽陰性を許容する安全側設計（特に当選/詐欺判定）。

## 3. 2軸の設計（A と B）← 当初からの基本方針
断層は「当選率の高さ」ではなく**完全自動が規約・技術・物理面で成立するか**で引く。

### ツールA（ほぼ完全自動・当選抽出型）
- 完全自動が合法・安全に成立する領域に限定して無人で量産：
  メール応募 / 公式API・LINE連携 / CAPTCHA無し＆禁止文言無しWebフォーム / **Web・会員型の即時当選（Xに近い手触りだが凍結リスク無し）** / 補助でアンケート・クイズ型。
- スパム/詐欺を排除し「本物の当選だけ」をメール/Discordに出力。
- 除外：X・Instagram・はがき・購入必須クローズド・CAPTCHA保護フォーム。

### ツールB（半自動・高当選率特化型）
- 当選率は高いが人手が要る/凍結リスクで自動化すべきでない領域：
  クローズド/レシート（約19%）/ Instagram投稿（約18.5%）/ 手動抽選の自治体・地方媒体 / X RT・フォロー / CAPTCHAフォーム。
- 収集・スコアリング・優先リスト・コメント案・入力補助まで自動、最終応募は人間。

### 振り分けルール
- CAPTCHA無し＆規約に自動応募禁止文言無し → A（自動）
- CAPTCHA有り or 規約で禁止 or 方式不明 → B（人間/エスカレーション）
- ポイ活系等はbot禁止規約が多いのでサイト個別の規約確認を必須化。

## 4. 当選率・利益率の重要知見（調査の結論）
- 手間がかかる方式ほど当たる：クローズド/レシート 約18.9%、Instagram投稿 約18.5%、ハガキ約14%、ネット約16.5%、**X拡散型は0.3〜1.5%（桁違いに低い）**。
- 協賛・店舗・地域限定は全国懸賞の約1.7倍当たりやすい。当選本数1,001名以上で約19%。
- **抽選方式の二分法（重要）**：即時当選（自動抽選）はタイミング技が運ゲー。手動抽選（自治体・中小・地方媒体）はタイミング（早期応募）と質（コメント・属性・ペルソナ）が当落に効く。普段懸賞をやらない自治体・企業は手動抽選が多い。
- **利益率の核心**：クローズド/レシートは「普段買う商品なら追加コスト0」で最も利益率が高いが、**懸賞のために買い増した瞬間に赤字化**（景表法：賞品額=取引額の20倍/上限10万、総額=売上2%）。無料系は薄利だが金銭リスク0で自動化向き。確定報酬モニターは時給換算で最高効率。
- 換金性：現金100% > 金券(JCB97/QUO88-95/Amazonギフト75-85%) > 物品。物品は小売価額の60%評価、金券は券面額。

## 5. ★ユーザーの嗜好・狙い（重要・忘れない）
- **所在地は広島。ただし全国狙いが主軸**（他地域の名産品・宿泊券が嬉しい）。居住地は補助。
- 観光協会の**常設アンケート懸賞**に最も興味（毎月当選・競争率低の穴場）。期間限定・新設サイトはさらに高ポテンシャル。
- スタンプラリーも含める（賞品郵送・在宅完結型を狙う）。
- 好みジャンル：**地酒・海産物・食品・特産品**（＋和牛・果物・米）。換金より「実用・体験」を重視する側面。
- 【是正方針 2026-06-09】換金性を重視しすぎて自治体・メーカー・市町村の懸賞が埋もれた/取れていない → 収集源にこれらを足し、スコアで実用・地域・手動抽選を浮上させる。

## 7. スコアリング設計
- 期待価値 = 当選確率係数 × 市場価値(log圧縮) × 換金率^cash_weight × 必要度 × 各種ボーナス。
- 利益率 roi = 期待価値 ÷ 応募コスト（cost.py）。rank_by で roi/score 切替。
- 加点：手間方式 / 手動抽選 / 協賛・地域限定(×1.7) / 当選本数多 / 開始直後 / 12月 / 属性一致 / **好みジャンル(preference)**。
- 確定報酬（モニター）は当選率=1.0。
- 【是正 2026-06-09】cash_weight で換金率の効きを弱め、manual_lottery/local/preference を強化して自治体・メーカー・特産・旅行を浮上。

## 8. 当選抽出（詐欺判定・ツールA中核）
- 必須ゲート：①応募ログ突合（未応募の当選は除外）②送信者ホワイトリスト完全一致（なりすまし@ID/ドメインは除外）。
- 即除外語：送料/手数料/先払い/クレカ番号/暗証番号。高リスク語複数で除外。
- メール認証 SPF/DKIM/DMARC=pass必須。青バッジは信頼根拠にしない。
- 本物だけ Discord 出力。受取期限リマインダーで失効防止。

## 9. 税務・換金（要点・要専門家確認）
- 一時所得：年50万円特別控除＋1/2課税。経費0なら年間当選評価額約90万円超で確定申告。住民税申告は別途。
- 反復・大規模は雑/事業所得化リスク。メルカリは2025/10改定で事業的転売制限。チケット/酒/医薬品は扱わない。
- 詳細は TAX_CASHOUT.md。

## 12. ★現在「自動」で動いている範囲（明確化）
- 自動：懸賞の収集→スコアリング→**Discordへの通知**（毎日）＋応募ログに基づく受取期限リマインダー。
- **動いていない**：自動応募（toola はワークフロー未登録＝意図的に手動）。当選メールの自動チェック（toolc run はIMAP情報未登録のため未稼働）。
- これは設計どおりの安全運用（A=自動の範囲は通知まで、応募は人間=B）。
- 自動範囲を広げる選択肢：(低リスク) 懸賞専用GmailのIMAPをSecrets登録→当選メール自動チェック。(高リスク・要規約確認) toola をワークフロー登録→メール/CAPTCHA無しフォームの自動応募。
